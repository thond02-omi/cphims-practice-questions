## Tasks 3.A.1–3.A.2 — SDLC and Project Management

> **[AP] RW-D3-01**: A hospital wants a new patient portal because "patients complain communication is hard." Before issuing an RFP, what should the project team do FIRST?
> - A) Select the portal with the most mobile features
> - B) Conduct needs analysis and current-state workflow assessment **[CORRECT]**
> - C) Negotiate an SLA
> - D) Begin user acceptance testing
>
> *Rationale*: Solution selection must begin with understanding the actual problem, current-state gaps, stakeholders, and functional requirements — not technology selection.
> *Maps to: Task 3.A.2*

> **[AN] RW-D3-02**: During scheduling system planning, clinicians request 50 enhancements. The go-live date is fixed because of a regulatory deadline. What should the project manager do?

>
> *Rationale*: MoSCoW (Must / Should / Could / Won't) separates go-live-critical requirements from items that can be deferred to a later phase without delaying the deadline.
> *Maps to: Task 3.A.2*

> **[AP] RW-D3-13**: A health system is building a new patient portal with frequently changing user requirements and clinician feedback cycles planned every two weeks. Which SDLC model is MOST appropriate?
> - A) Waterfall, because it provides clear phase gates and formal documentation

> **[AN] RW-D3-28**: An informatics director reviews a project charter for a new scheduling system and notices it lists the project objectives, sponsor name, and budget, but does not define the project scope or list key stakeholders. What SDLC artifact should be updated NEXT?

> *Maps to: Task 3.A.1, 3.A.2*

## Task 3.A.3 — Process Improvement Concepts

> **[RE] RW-D3-14**: A hospital quality team is reducing CAUTI infections. They quantify baseline infection rates, identify contributing root causes through data analysis, implement catheter care protocol changes, and establish ongoing monitoring dashboards. This structured methodology is:
> - A) PDCA
> - B) ADKAR
> - C) DMAIC **[CORRECT]**
> - D) Kotter's 8-Step Model
>
> *Rationale*: DMAIC (Define, Measure, Analyze, Improve, Control) is the Six Sigma process improvement framework. PDCA is similar but uses lighter statistical rigor.
> *Maps to: Task 3.A.3*

> **[AN] RW-D3-26**: A clinical informatics team observes that a 12-step medication ordering workflow contains only 4 steps that directly contribute to patient safety verification; the other 8 involve re-entering data, redundant approvals, and searching for duplicate information. Which improvement approach BEST identifies and eliminates these non-value-added steps?
> - A) DMAIC
> - B) PDCA
> - C) Lean / value stream mapping **[CORRECT]**
> - D) Fishbone analysis
>
> *Rationale*: Lean focuses on eliminating waste (non-value-added steps); value stream mapping visualizes the entire workflow to surface waste and inefficiency.
> *Maps to: Task 3.A.3*

> **[AP] RW-D3-29**: A quality improvement team in a surgical department repeatedly asks "why" to trace a high post-operative complication rate from its observable symptom back to a root operational cause. After five iterations, they identify inadequate surgeon briefing as the root cause. Which technique is being applied?
> - A) PDCA
> - B) Affinity diagram
> - C) 5 Whys **[CORRECT]**
> - D) SIPOC
>
> *Rationale*: The 5 Whys technique iteratively asks "why?" to drill through symptoms to the underlying root cause. It is a Lean/Six Sigma root cause analysis tool.
> *Maps to: Task 3.A.3*

## Task 3.A.4 — Visualization Tools for Process Analysis

> **[AP] RW-D3-15**: An implementation team is mapping the hospital discharge process and needs to show which staff role — case manager, physician, nurse, or social worker — performs each step and where handoffs occur. The MOST appropriate tool is:
> - A) Fishbone diagram
> - B) Swim lane diagram **[CORRECT]**
> - C) Run chart
> - D) SIPOC diagram

> *Rationale*: Swim lane diagrams assign each role its own lane and show exactly who performs each step and where cross-role handoffs occur.
> *Maps to: Task 3.A.4*

> **[AP] RW-D3-41**: A project team is scoping a new prior authorization workflow and needs a high-level view showing the suppliers, inputs, key process steps, outputs, and customers before doing detailed process mapping. The MOST appropriate tool is:
> - A) Swim lane diagram
> - B) Fishbone diagram
> - C) SIPOC diagram **[CORRECT]**
> - D) Control chart
>
> *Rationale*: SIPOC (Supplier-Input-Process-Output-Customer) is a high-level scoping tool used at the start of a process improvement initiative to define boundaries before detailed mapping.
> *Maps to: Task 3.A.4*

> **[AP] RW-D3-42**: After a series of medication errors in the ICU, the patient safety officer wants to visualize all potential contributing causes organized by categories such as People, Process, Technology, Environment, and Policy. The MOST appropriate tool is:
> - A) Swim lane diagram

> - C) Fishbone (Ishikawa) diagram **[CORRECT]**

## Tasks 3.A.5–3.A.10 — Additional Business Analysis Questions

> **[AP] RW-D3-48**: A hospital implements a new sepsis early-warning system at a cost of $200K. Twelve months later, sepsis-related ICU admissions dropped by 30, each avoided case saving approximately $15,000 in costs. What is the first-year ROI?
> - A) 30%
> - B) 125% **[CORRECT]**
> - C) 15%
> - D) 200%
>
